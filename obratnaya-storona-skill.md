# Обратная сторона — скилл проекта

## Суть

Сайт-визитка независимого медиа «Обратная сторона» о литературе, визуальном искусстве и культурных инициативах. Главный образ: предметы культуры движутся по орбите и исчезают в чёрной дыре в полу комнаты.

Тон: спокойный, интеллектуальный, минималистичный. Не делать сайт похожим на лендинг или интерфейс SaaS.

## Визуальная система

- Светлая белая комната с углом, бетонным полом и чёрным овальным отверстием.
- Чёрная дыра остаётся глубокой, гладкой и контрастной, без видимых пикселей.
- В сцене используются: картина Ван Гога, портрет писателя, вырезанная статуя Родена и газета The Times.
- Предметы не падают вертикально: они вращаются по большой траектории, ускоряются у дыры и постепенно исчезают.
- Навигация состоит из чёрных точек, подписей и тонких соединяющих линий.
- В мобильной версии не использовать видимый круг, карточку или плотную подложку вокруг точек: только точки и связи поверх сцены.

## Разделы навигации

| Раздел | Смысл |
| --- | --- |
| Сигнал | Лонгриды, рецензии и события, заметные с обратной стороны. |
| Мерцание | Визуальные материалы, подборки, фрагменты и малые формы. |
| Жители | Интервью, голоса и фигуры культурной среды. |
| Витрина | Книги, обложки и предметы будущего магазина. |
| Слово дня | Ежедневная словарная находка с коротким комментарием. |
| Архив | Следы, документы и будущие материалы издания. |

## Техническая архитектура

- Vite + JavaScript + Three.js.
- Основная сцена и интерактивная навигация: `src/main.js`.
- Стили и адаптивные правила: `src/styles.css`.
- Публичные изображения: `public/assets/`.
- Деплой GitHub Pages: `.github/workflows/deploy.yml`.
- В `index.html` обязательно сохранять:

```html
<meta name="viewport" content="width=device-width, initial-scale=1" />
```

Без этого iPhone Chrome открывает сайт в виртуальной десктопной ширине, и мобильная компоновка перестаёт работать корректно.

## Правила для телефона

- Проверять в размере, близком к iPhone 16 Pro: `402 x 874` CSS-пикселей.
- Точки должны быть тонкими. Активная точка может быть немного больше, но не должна выглядеть как крупная кнопка.
- Подписи точек не должны пересекать друг друга или чёрную дыру.
- При касании точки появляется название раздела и описание под чёрной частью дыры, по центру.
- Когда открыт текст раздела, нижний футер скрывается, чтобы не пересекаться с описанием.
- В мобильном режиме фоновая сцена не принимает касания; управляется только верхний слой с точками.
- Жест двумя пальцами увеличивает сеть точек поверх сцены. При уменьшении жестом сеть возвращается в исходную позицию.

## Перед любым изменением

1. Не менять десктопный конус навигации, если задача касается только телефона.
2. Не менять положение точек, когда просят изменить только текстовый блок.
3. После правок проверять минимум три размера: `390 x 667`, `402 x 874`, `768 x 1024`.
4. Проверять, что выбор точки работает, а текст не пересекается с футером.
5. Выполнить `npm run build`.
6. Сделать скриншот мобильного состояния до публикации.

## Команды

```bash
npm run dev
npm run build
```

Локальный адрес: `http://127.0.0.1:5173/`.

## Публикация

1. Проверить сборку.
2. Закоммитить только относящиеся к задаче файлы.
3. Отправить изменения в ветку `main`.
4. Дождаться успешного GitHub Actions workflow `deploy.yml`.
5. Проверить публичную страницу: https://nikiau12.github.io/obratnaya-storona/
